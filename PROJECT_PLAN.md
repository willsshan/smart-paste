# Windows 11 剪切板增强工具项目方案

## 1. 项目定位

这是一个面向 Windows 11 的轻量级剪切板增强工具，核心目标是：

- 比系统剪切板历史更快
- 键盘操作更顺手
- UI 精致且不过度打扰
- 常驻内存小、响应快、启动快

一句话定义：

> 一个以“粘贴效率”为核心的、偏键盘驱动的 Windows 11 剪切板管理器。

---

## 2. 你的核心想法，建议这样定义

你提到的核心交互非常好，可以整理成一个清晰的主流程：

1. 用户平时正常复制内容，程序自动记录剪切板历史。
2. 用户在任意输入框中按 `Ctrl+V`。
3. 默认先粘贴最近一次复制的内容。
4. 如果用户继续按 `上/下方向键`，就在一个轻量浮层或小面板里切换最近历史项。
5. 按 `Enter` 确认粘贴当前项，按 `Esc` 退出。

这个思路本质上是：

- `Ctrl+V` 仍然符合用户习惯
- 但在“粘错 / 想切换到前几条内容”时，能无缝进入历史选择

这是很强的产品点，建议作为项目主卖点。

---

## 3. 功能设计

## 3.1 MVP 核心功能

第一版建议先做这些，足够形成可用产品：

- 剪切板历史监听
- 文本内容存储
- `Ctrl+V` 触发粘贴最近项
- 粘贴后短时间内支持 `上/下键` 切换历史项
- 小型悬浮 UI，显示最近 5~20 条历史
- 回车确认、Esc 关闭
- 开机自启动
- 系统托盘图标
- 历史清空
- 历史条数限制，例如 100~300 条

## 3.2 强化功能

MVP 稳定后，第二阶段建议加入：

- 图片剪切板支持
- 文件列表复制支持
- 富文本 / HTML 支持
- 历史搜索
- 固定收藏项（Pin）
- 最近常用片段置顶
- 去重策略
- 应用黑名单

说明：

- 图片支持：截图后可快速再次粘贴
- 文件列表支持：资源管理器里复制文件后可再次粘贴路径或文件对象
- 富文本支持：在 Word、微信、Outlook 等场景体验更完整
- 应用黑名单：例如密码管理器、银行软件、远程桌面等不记录

## 3.3 高价值发散功能

这些不是必须，但很容易把产品做出差异化：

- 文本片段库（Snippets）
- AI 清洗粘贴
- 粘贴变体
- 快捷动作
- 多槽位临时寄存
- 分组历史
- 云同步（后期）

可展开为：

- `纯文本粘贴`：去掉格式再粘贴
- `一键去换行`
- `一键去多余空格`
- `中英文标点统一`
- `代码粘贴模式`
- `邮箱/手机号脱敏后粘贴`
- `翻译后粘贴`

如果你以后想做成更强的效率工具，这部分会非常有潜力。

## 4. 交互设计建议

## 4.1 不建议只靠“按 Ctrl+V 后再按上下键”硬做

原因是：

- 很多软件会自己消费方向键
- 方向键是否能被你稳定拦截，取决于全局键盘 Hook 和前台窗口时机
- 如果直接替换系统 `Ctrl+V` 行为，兼容性会比较敏感

更稳妥的交互建议是双通道：

### 通道 A：保留你的主设计

- 用户按 `Ctrl+V`
- 先快速粘贴最近项
- 在一个极短时间窗口内（如 800ms~1500ms）监听 `上/下键`
- 如果用户按了方向键，弹出小型历史切换 UI

适合追求“几乎不改变原习惯”。

### 通道 B：增加显式唤起快捷键

- 例如 `Ctrl+Shift+V` 或 `Alt+V`
- 直接打开历史面板
- 方向键切换，Enter 粘贴

这条路径更稳定，也更容易被新用户理解。

最佳实践是：

- `Ctrl+V` 做极速最近粘贴
- `Ctrl+Shift+V` 做完整历史选择

这样兼顾流畅和兼容性。

## 4.2 建议 UI 形态

推荐使用：

- 小型无边框浮窗
- 半透明亚克力 / Mica 风格
- 圆角
- 阴影柔和
- 动画时长控制在 120ms~180ms

面板内容建议：

- 当前选中项大一点、对比更强
- 每条历史显示：
  - 内容预览
  - 类型图标（文本 / 图片 / 文件）
  - 来源应用图标或名称
  - 时间
- 支持键盘高亮移动

Windows 11 风格关键词：

- Mica / Acrylic
- Fluent Design
- WinUI 风格圆角与层级
- 不要做得像 Electron 的普通网页弹窗

## 5. 推荐技术架构

## 5.1 最推荐方案

如果你的目标是：

- Windows 11 原生体验
- 内存占用小
- 性能好
- 动画顺滑
- 全局快捷键和系统集成更容易

我最推荐：

### `Rust + Tauri + 原生 Windows API`

职责拆分：

- `Rust Core`
  - 剪切板监听
  - 历史存储
  - 热键监听
  - 粘贴注入
  - 数据清洗与策略
- `Tauri Shell`
  - 窗口管理
  - 前后端通信
  - 托盘
  - 配置管理
- `前端 UI`
  - 建议 `SolidJS` 或 `Svelte`
  - 做轻量、动画流畅的历史面板

这个组合的优点：

- 比 Electron 小很多
- 内存明显更友好
- 启动更快
- Rust 非常适合做常驻后台服务、事件监听、缓存管理
- 后期加数据库、图片处理、索引搜索也方便

## 5.2 为什么不优先推荐 Electron

Electron 也能做，但对你这个项目不算最佳：

- 常驻内存通常更高
- 启动速度和后台资源占用吃亏
- 全局输入、剪切板、系统级交互虽然可做，但不如原生/Rust 路线舒服

如果你明确想快速出界面 Demo，Electron 可以做原型；
如果你想做一个长期可用、轻量、像真正 Windows 工具的软件，优先 Rust + Tauri。

## 5.3 如果你更偏微软生态

第二推荐方案：

### `C# + WinUI 3 + Windows App SDK`

优点：

- Windows 11 原生 UI 非常合适
- 系统 API 调用顺手
- 开发效率高
- UI 一致性强

缺点：

- 常驻内存通常不会比 Rust 更优
- 有些底层控制和性能极致优化空间略弱于 Rust

结论：

- 想要“最快做出漂亮 Windows 工具”：`C# + WinUI 3`
- 想要“更轻、更强、更适合长期打磨”：`Rust + Tauri`

## 6. 模块划分

建议把项目拆成以下模块：

## 6.1 Core 核心层

负责：

- 剪切板监听
- 数据标准化
- 历史入库
- 去重和过期淘汰
- 敏感内容过滤

建议数据结构：

- `ClipboardItem`
  - `id`
  - `type`
  - `content`
  - `preview`
  - `source_app`
  - `created_at`
  - `is_pinned`
  - `hash`

## 6.2 Hotkey / Input 层

负责：

- 全局快捷键注册
- `Ctrl+V` 增强逻辑
- `Ctrl+Shift+V` 历史面板唤起
- 键盘导航
- 模拟粘贴

注意：

- “模拟粘贴”通常需要注入键盘事件
- 某些 UAC 提权窗口、沙盒软件、游戏内输入框可能有限制

## 6.3 UI 层

负责：

- 历史面板
- 搜索框
- 设置页
- 托盘菜单
- 首次引导

## 6.4 Storage 层

负责：

- 本地持久化
- 历史索引
- 配置管理

建议：

- 配置：`JSON` / `TOML`
- 历史库：`SQLite`

SQLite 足够了，优点：

- 小
- 稳定
- 支持搜索和筛选
- 方便以后做统计和同步

## 6.5 Platform Adapter 层

负责封装 Windows 特有能力：

- 剪切板 API
- 全局键盘 Hook
- 热键注册
- 窗口定位
- 托盘
- 开机自启

这样后面如果要支持 macOS，核心层能复用一部分。

## 7. 数据与性能策略

这是这个项目非常关键的部分。

## 7.1 内存控制建议

- 常驻内存目标：`30MB~80MB`
- 历史列表只把预览数据常驻内存
- 大文本、图片、HTML 原文按需加载
- 图片缩略图缓存，不把完整位图长期放内存
- 历史记录限制条数和总大小

建议策略：

- 文本直接入 SQLite
- 图片保存到本地缓存目录
- 数据库里只存路径、尺寸、预览信息

## 7.2 去重策略

建议默认开启“短时间去重”：

- 同样内容重复复制，不新增历史
- 但更新时间和最近使用排序

否则用户连续复制同一段内容时，历史会被刷满。

## 7.3 搜索策略

文本搜索建议支持：

- 前缀匹配
- 模糊包含
- 拼音首字母匹配（如果你想更适合中文用户）

中文场景下，拼音搜索会是一个很强的亮点。

## 7.4 流畅性建议

- UI 展示只渲染当前可见项
- 列表虚拟化
- 动画尽量使用 transform / opacity
- 后台任务异步化
- 不要在 UI 线程做剪切板解析和图片处理

---

## 8. 隐私与安全

这个项目必须重视。

建议默认就做：

- 敏感应用黑名单
- 隐私模式开关
- 一键暂停监听
- 自动清理历史
- 本地数据加密（至少加密敏感文本）

高风险内容建议不记录：

- 密码输入框来源
- 密码管理器
- 银行/支付类应用
- 远程桌面会话内特定场景

你甚至可以加一个功能：

- 检测类似密码格式的短文本，提醒是否忽略入库

## 9. 你可能没想到但很重要的点

## 9.1 来源应用识别

记录“这条内容从哪个应用复制出来”很有用：

- 用户更容易找到内容
- UI 更高级
- 方便后续做过滤

例如显示：

- VS Code
- 微信
- Edge
- Word

## 9.2 粘贴模式差异

不是所有输入框都一样。

至少会遇到：

- 普通文本框
- 富文本编辑器
- 浏览器输入框
- 终端
- 代码编辑器
- 管理员权限窗口

所以粘贴实现最好设计成多策略回退：

1. 优先原生剪切板注入 + 粘贴快捷键
2. 不行则模拟按键
3. 特殊场景再做兼容处理

## 9.3 首次教育

这个产品有一定学习成本，所以第一次启动建议告诉用户：

- `Ctrl+V` 还是正常粘贴
- `Ctrl+Shift+V` 可查看历史
- `↑/↓` 可切换
- `Enter` 确认
- `Esc` 退出

## 9.4 设置项不要太多

效率工具最怕设置爆炸。

设置页建议分三组：

- 基础：开机启动、历史数量、主题
- 粘贴：快捷键、纯文本粘贴、去重
- 隐私：黑名单、自动清理、暂停监听

## 11. 具体技术选型建议

## 方案 A：长期最佳

- 核心语言：`Rust`
- 桌面壳：`Tauri`
- UI：`SolidJS + Tailwind` 或 `Svelte + Tailwind`
- 本地数据库：`SQLite`
- Windows API：`windows-rs`
- 全局热键/Hook：优先自己封装，必要时选成熟库

适合：

- 想做成真正产品
- 重视性能、内存、细节

## 方案 B：开发效率最佳

- 语言：`C#`
- UI：`WinUI 3`
- 数据：`SQLite`
- 系统能力：Windows App SDK + Win32 API 补充

适合：

- 想快速开发
- 团队更熟悉 .NET
- 优先 Windows 原生体验

如果这是你准备认真做的项目，我建议这样定：
